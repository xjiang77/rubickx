Learning Engineering for the Chinese-first web

把网上经典学习材料，变成可复用、可迭代、可评测的中文学习资产。

rubickx 的目标不是再造一个资料收藏夹，而是建立一条完整工作流： 从 GitHub repo、podcast、YouTube video 到中文摘要、interactive course、blog post，再到可重复运行的 harness evaluation。

Current Loop

  1. Collect timeless learning resources
  2. Extract durable concepts and trade-offs
  3. Write Chinese-first summaries
  4. Transform into course or article when needed
  5. Grade outputs with a deterministic harness
Primary language

Chinese explanation, English terms preserved where they matter.

Mission

不是做“内容搬运”，而是做“学习重构”。

Input

经典材料来源于真实互联网：优秀 repo、技术播客、视频课程、长文档、reference book。

Transformation

每份材料都先判断适不适合重构，再决定输出为中文摘要、interactive course，或 blog post。

Evaluation

输出不是“看起来不错”就算完成，而是进入 harness，用 grounding、 pedagogy、traceability 等标准评分。

Project Tracks

rubickx 同时做两件事：学 agent，也学怎么教人学习。

Track A

Agent Tutorial Core

基于 learn-claude-code 的多语言实现，当前重点是 Go 版本的 12 个递进式 session。

  • 从 agent loop 到 worktree task isolation
  • 保留 tutorial 结构，便于系统理解 agent mechanism
  • 中文 walkthrough 和英文文档并存
Track B

Content Forge

面向真实学习材料构建“收集、判断、转化、评分”的内容生产管线。

  • 经典资源优先，而不是追热点
  • Chinese-first 输出，保留关键 English terminology
  • 最终目标是沉淀可复用学习资产库

Harness Engineering

用可执行评测，而不是口头标准，来逼近高质量学习内容。

make harness-list
make harness-init RUN=harness/runs/demo
make harness-grade RUN=harness/runs/demo

What gets graded

Grounding
输出是否真的 anchored 在 source material。
Decision Fit
这份材料到底该做 course 还是 blog，判断是否合理。
Chinese Output
是否真正做到中文优先，而不是英文摘要加少量翻译。
Traceability
失败时能否从 trace 里看见过程，而不是只看到坏结果。

Why It Matters

经典材料很多，但真正被“学会”的很少。

“资料收藏”解决的是获取问题；rubickx 想解决的是理解、重构、复用和验证的问题。

对 AI agent 而言，最有价值的不是写出一篇漂亮总结，而是能稳定地产出适合学习的内容。
